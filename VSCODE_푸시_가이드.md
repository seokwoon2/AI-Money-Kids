# 🚀 VS Code에서 푸시하기 (가장 확실한 방법)

## 단계별 가이드

### 방법 1: 소스 제어 패널 사용 (가장 쉬움!)

**1단계: 소스 제어 열기**
- VS Code 왼쪽 사이드바에서 **소스 제어 아이콘** 클릭
  - 아이콘 모양: 분기 모양 또는 "1" 숫자가 표시됨
  - 또는 키보드: `Ctrl + Shift + G`

**2단계: 변경사항 확인**
- "Changes" 섹션에서 변경된 파일 목록 확인
- 모든 파일 옆에 "+" 아이콘이 있으면 자동으로 추가됨

**3단계: 커밋 메시지 입력**
- 위쪽 "Message" 입력란에 커밋 메시지 입력
  - 예: `Update: Gemini API and Streamlit Cloud secrets`
  - 또는: `Update: 최신 코드 반영`

**4단계: 커밋**
- 위쪽 **"✓" 아이콘** 클릭 (또는 `Ctrl + Enter`)
- "Commit" 버튼 클릭

**5단계: 푸시**
- 위쪽 **"..." (점 3개)** 메뉴 클릭
- **"Push"** 선택
- GitHub 로그인 창이 열리면 로그인

---

### 방법 2: 상태바에서 푸시 (더 빠름!)

**1단계: 상태바 확인**
- VS Code 하단 상태바 확인
- 왼쪽에 **"main"** 또는 브랜치 이름이 보임

**2단계: 브랜치 클릭**
- **"main"** 클릭

**3단계: 푸시 선택**
- 메뉴에서 **"Push"** 선택
- 또는 **"Push to..."** 선택 후 원격 저장소 선택

---

### 방법 3: 명령 팔레트 사용

**1단계: 명령 팔레트 열기**
- `Ctrl + Shift + P` 누르기

**2단계: Git Push 입력**
- "Git: Push" 입력
- Enter 키 누르기

---

## 문제 해결

### "Push" 버튼이 비활성화되어 있을 때:
- 먼저 커밋이 필요합니다
- 위쪽 "Message" 입력란에 메시지 입력 후 "✓" 클릭

### GitHub 로그인 창이 안 열릴 때:
- 브라우저에서 직접 GitHub 로그인: https://github.com/login
- VS Code에서 GitHub 계정 연결 확인

### 네트워크 오류가 날 때:
- 인터넷 연결 확인
- GitHub 접속 가능한지 확인: https://github.com
- 잠시 후 다시 시도
